THIS IS A F***ING WEBSITE

See? This is a f***ing website, with posts, cool texts and a good-looking page. It’s simple, readable and the client who chooses what specific font to display. Only with a few lines of CSS and using JavaScript just so it can be readable on both mobile and desktop. Why would I need more than that just to write things that people can read? I mean, web browsers were created so we can share content via HTML.

Why would I need to specify a custom 20Mb font from google fonts if you can just say what variant the client should render? Choices are cool and the user should have the liberty to choose what he/she finds as the most readable font.

Also, why the fook you page is basically just JavaScript? I mean, content is done with HTML. I know that your JS is “automagically” setting content when the user opens the page but my dear why the fook you want to do things the most difficult way?.

See my page? I can do everything we should be doing in the web: I can post readable articles, maintain a simple list of them but don’t waste user’s time. The page loads instantly because it’s a static HTML page with a brief stylesheet and just a fair use of JavaScript. Why would I do anything beside that? It just works and has all features that matter.

If you, webdev do all that fancy stuff just to show off, you should just get off the web because you’re wasting all of my and your time. WHY THE FOOK YOU REALLY NEED FANCY ANIMATIONS, HEAVY FONTS AND BUTTONS THAT CHANGE A TOTAL OF NOTHING IN THE USER EXPERIENCE?.

Look at this article. I wrote it in Markdown and them runned a 70 lines Python script to generate a page and add it to the articles list. The simplest and most functional static site generator you’ll ever see. Loaded in milisseconds. Didn’t wasted my or your time.

And if you still wants some more blogish cOlL features you could just use another static generator like hugo. Come on, you’ll like it. Using Markdown to write articles is cool, you can just write what you want to say and hugo takes care of the style.

Don’t be cringe, webdev! Let’s make web great again!
